GitHub содержит миллионы проектов — от маленьких скриптов до больших фреймворков. Если вы хотите использовать чужой код или изучить исходные тексты, нужно скачать проект на свой компьютер. В этом гайде я покажу вам 4 способа это сделать, от самого простого до профессионального.

## Введение: 4 способа скачать проект с GitHub

Каждый способ подходит для разных ситуаций. Выберите то, что вам нужно:

| Способ | Команда | Используется для | Требует git? |
|--------|---------|-----------------|----------|
| **git clone** | `git clone https://...` | Полный проект с историей | Да |
| **ZIP-архив** | Кнопка Download ZIP | Одноразовое использование | Нет |
| **GitHub CLI** | `gh repo clone user/repo` | Быстрое клонирование | Да |
| **Отдельный файл** | Кнопка Raw или `curl` | Один файл для изучения | Нет |

Начнём с самого универсального способа — **git clone**.

## Способ 1: git clone (рекомендуемый — полный репозиторий с историей)

Это самый популярный способ. Команда `git clone` скачивает весь проект, включая всю историю изменений. После этого вы можете обновлять код, просматривать коммиты и даже предлагать свои изменения.

### HTTPS: самый простой способ

Откройте терминал (командную строку) и введите:

```bash
git clone https://github.com/user/repo.git
```

Замените `user` на имя автора и `repo` на название проекта. Например:

```bash
git clone https://github.com/torvalds/linux.git
```

После этого в текущей папке появится папка `linux` со всем содержимым проекта.

### SSH: безопаснее, но нужна настройка

Если вы уже настроили SSH-ключи, можете использовать более безопасный способ:

```bash
git clone git@github.com:user/repo.git
```

SSH-ключи удобнее, потому что не нужно вводить пароль каждый раз. Если вы не знаете, что это такое, используйте HTTPS.

### Клонировать в конкретную папку

По умолчанию проект скачивается в папку с названием репозитория. Если вы хотите другое имя:

```bash
git clone https://github.com/user/repo.git my-project-folder
```

Проект скачается в папку `my-project-folder`.

### Где найти URL для клонирования на GitHub

1. Откройте страницу проекта на GitHub
2. Нажмите зелёную кнопку **Code** (в правой части, над файлами)
3. Выберите **HTTPS** или **SSH** (зависит от вашей настройки)
4. Скопируйте ссылку в буфер обмена
5. Вставьте её в команду `git clone`

Если вы видите только HTTPS — это нормально, используйте его.

## Способ 2: Скачать как ZIP-архив (без git, без истории)

Не хотите устанавливать git? Нет проблем. GitHub может отправить весь проект одним ZIP-файлом.

### Как скачать через браузер

1. Откройте страницу проекта на GitHub
2. Нажмите **Code** → **Download ZIP**
3. Браузер скачает файл (обычно называется `repo-main.zip`)
4. Распакуйте архив в нужную папку

Готово! Файлы проекта на вашем компьютере.

### Когда ZIP лучше, чем git clone?

- **Одноразовое использование** — вам просто нужно посмотреть код
- **Нет интернета** — скачали, работаете офлайн
- **git не установлен** — не хотите возиться с установкой
- **Простота** — не нужно знать команды терминала

### Недостатки ZIP-архива

⚠️ **Нет истории** — вы не видите, как менялся код

⚠️ **Нельзя обновляться** — когда автор выпустит новую версию, придётся скачивать заново

⚠️ **Нельзя отправлять изменения** — даже если вы что-то улучшили, без git-репозитория не поделитесь с автором

Если эти ограничения вас не беспокоят — используйте ZIP.

## Способ 3: GitHub CLI (`gh repo clone`)

GitHub имеет свой инструмент командной строки — GitHub CLI. Это мощнее, чем обычный git, и удобнее для работы с GitHub.

### Установка GitHub CLI

Сначала установите инструмент:

- **macOS**: `brew install gh`
- **Windows**: `choco install gh` (если установлен Chocolatey)
- **Linux**: смотрите на https://cli.github.com/

### Использование

После установки используйте:

```bash
gh repo clone user/repo
```

Например:

```bash
gh repo clone microsoft/vscode
```

GitHub CLI автоматически выберет лучший способ подключения (HTTPS или SSH) и скачает проект.

### Преимущества перед браузером и обычным git

✅ **Автоматическая авторизация** — вам не нужно вводить пароль

✅ **Меньше команд** — не нужно вспоминать HTTPS или SSH

✅ **Интеграция** — можно работать с Issues, Pull Requests прямо из терминала

✅ **Удобнее для приватных репозиториев** — автоматически авторизуется

## Способ 4: Скачать отдельный файл

Иногда вам нужен не весь проект, а только один файл. Например, одна функция или конфиг.

### Через браузер (кнопка Raw)

1. Откройте файл на GitHub
2. Нажмите кнопку **Raw** (справа вверху)
3. Браузер откроет файл в виде текста
4. Нажмите Ctrl+S (Cmd+S на Mac) и сохраните

### Через команду curl

Если вы знаете прямую ссылку на файл:

```bash
curl -O https://raw.githubusercontent.com/user/repo/main/file.txt
```

Файл скачается в текущую папку.

### Через команду wget

Альтернатива для Linux:

```bash
wget https://raw.githubusercontent.com/user/repo/main/file.txt
```

Оба способа одинаковые, просто разные инструменты.

## Клонирование приватного репозитория

Если проект приватный и вам дан доступ, нужна авторизация.

### HTTPS + Personal Access Token

1. На GitHub откройте **Settings** → **Developer settings** → **Personal access tokens**
2. Нажмите **Generate new token**
3. Дайте название, выберите `repo` и подтвердите
4. Скопируйте токен (он покажется только один раз!)
5. Используйте при клонировании:

```bash
git clone https://YOUR_TOKEN@github.com/user/private-repo.git
```

Замените `YOUR_TOKEN` на скопированный код.

⚠️ **Не публикуйте токен** в интернете и в коде!

### SSH-ключи

Это безопаснее, но требует настройки. Если вам интересно, прочитайте отдельную статью по этой теме. GitHub CLI и обычный git с SSH — самый надёжный вариант для постоянной работы.

### GitHub CLI автоматически авторизуется

Самый простой способ для приватных репозиториев:

```bash
gh repo clone user/private-repo
```

GitHub CLI уже знает ваши учётные данные, нужно только один раз авторизоваться через команду `gh auth login`.

## Клонирование конкретной ветки или тега

Проекты часто имеют несколько веток (ветвей) — одна для стабильной версии, другие для разработки. Вы можете клонировать конкретную ветку.

### Клонировать ветку (branch)

```bash
git clone --branch develop https://github.com/user/repo.git
```

Вместо `develop` напишите название нужной вам ветки. Обычно есть:
- `main` или `master` — стабильная версия
- `develop` — версия в разработке
- другие названия — экспериментальные ветки

### Клонировать конкретный релиз (tag)

```bash
git clone --branch v2.0 https://github.com/user/repo.git
```

Теги — это метки для важных версий. Обычно пишутся как `v1.0`, `v2.0.1` и так далее.

Если вы не знаете, какую ветку выбрать, клонируйте без `--branch` — будет `main`.

## Shallow clone для экономии места и времени

Если проект очень большой (например, репозиторий Linux с историей 30 лет), клонирование может занять долго и места на диске. Решение — **shallow clone**:

```bash
git clone --depth 1 https://github.com/user/repo.git
```

Параметр `--depth 1` скачивает только последний коммит, без истории. Это экономит время и место в 10+ раз.

### Когда это используется

✅ **CI/CD** — сервис автоматической проверки и развёртывания

✅ **Docker контейнеры** — скачиваем только нужное, не весь архив

✅ **Большие проекты** — избегаем долгого клонирования

✅ **Быстрая работа** — нам не нужна история, только текущий код

### Ограничения

⚠️ Вы не можете видеть историю коммитов

⚠️ Некоторые операции git работают медленнее

Если позже вам нужна вся история, команда `git fetch --depth=2147483647` их подтянет.

## Часто задаваемые вопросы (FAQ)

### Как скачать проект с GitHub без git?

Используйте **ZIP-архив** — самый простой способ. Нажмите Code → Download ZIP и распакуйте. Альтернатива — GitHub CLI, но его тоже нужно установить.

### В чём разница между git clone и скачать ZIP?

| | git clone | ZIP |
|--|-----------|-----|
| История | Вся история коммитов | Только текущая версия |
| Размер | Может быть больше | Компактнее |
| Обновление | `git pull` обновляет | Нужно скачивать заново |
| Возможность отправить изменения | Да, через pull request | Нет |
| Требует знание команд | Да | Нет |

Выбирайте git clone, если планируете работать с проектом долго.

### Как скачать приватный репозиторий GitHub?

1. Убедитесь, что вам дан доступ
2. Используйте GitHub CLI (`gh repo clone user/repo`) — самый простой способ
3. Или используйте HTTPS с Personal Access Token (смотрите раздел выше)
4. Или настройте SSH-ключи

GitHub CLI — рекомендуемый способ для приватных проектов.

### Как скачать только одну папку с GitHub?

К сожалению, стандартный git clone не позволяет скачать только одну папку. Варианты:

**Вариант 1** — скачайте весь проект и удалите ненужные папки

**Вариант 2** — используйте git sparse-checkout (продвинутая функция):

```bash
git clone --filter=blob:none --sparse https://github.com/user/repo.git
cd repo
git sparse-checkout add path/to/folder
```

**Вариант 3** — скачайте ZIP и распакуйте только нужную папку

Для большинства случаев вариант 1 — самый простой.

### Как обновить скачанный репозиторий?

Если вы клонировали через `git clone`, обновление простое:

```bash
cd my-project
git pull
```

Команда `git pull` скачает все новые изменения, которые добавил автор.

**Если вы скачали ZIP** — придётся скачивать ZIP заново.

### Нужно ли устанавливать git для скачивания с GitHub?

**Нет, не обязательно.**

- **Для ZIP-архива** — не нужен git
- **Для отдельных файлов** — не нужен git
- **Для GitHub CLI** — нужна сама CLI, но не обязателен git
- **Для git clone** — нужен git

Но я рекомендую установить git всё равно. Это стандарт в разработке, и он вам пригодится.

Теперь вы знаете 4 способа скачать проект с GitHub:

1. **git clone** — универсальный способ, полный проект с историей

3. **GitHub CLI** — удобный инструмент для профессионалов
4. **Отдельный файл** — когда нужен только один файл
