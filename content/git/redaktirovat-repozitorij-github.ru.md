GitHub позволяет редактировать файлы прямо в браузере — без установки Git, без клонирования репозитория. Это удобно для небольших правок: исправить опечатку в README, обновить конфигурационный файл, добавить документацию. Для серьёзной разработки с несколькими файлами лучше подходит локальная работа или GitHub Codespaces.

## Редактирование существующего файла

Самый простой способ — открыть файл и нажать иконку карандаша:

```
1. Открыть репозиторий на GitHub

2. Перейти к нужному файлу (например, README.md)

3. Нажать иконку карандаша (Edit this file) в правом верхнем углу
   при просмотре файла

4. Внести изменения в редакторе

5. В разделе "Commit changes" внизу:
   - Написать сообщение коммита
   - Выбрать ветку (текущую или создать новую)

6. Нажать "Commit changes"
```

Файл изменён — новый коммит появился в истории репозитория.

## Горячие клавиши в редакторе GitHub

Веб-редактор поддерживает часть горячих клавиш:

```
Ctrl+Z (Cmd+Z)     — отменить последнее действие
Ctrl+Y (Cmd+Y)     — повторить отменённое
Ctrl+F (Cmd+F)     — поиск в файле
Ctrl+H             — поиск и замена
Tab                — отступ (настраивается)
```

Для более удобного редактирования можно открыть GitHub в режиме редактора кода — нажать клавишу `.` (точку) на странице репозитория или изменить URL с `github.com` на `github.dev`.

## Создание нового файла

Можно создать файл не выходя из браузера:

```
1. Открыть репозиторий

2. Нажать кнопку "Add file" → "Create new file"

3. Ввести имя файла (включая путь)
   Например: docs/setup.md
   Если написать "docs/", GitHub автоматически создаст папку

4. Написать содержимое файла

5. Заполнить сообщение коммита

6. Нажать "Commit new file"
```

Создание вложенных папок: просто указать путь в имени файла. `docs/api/endpoints.md` создаст файл `endpoints.md` внутри `docs/api/`.

## Загрузка файлов через веб-интерфейс

GitHub позволяет загружать файлы через браузер (drag & drop или диалог выбора):

```
1. Открыть нужную папку в репозитории

2. Нажать "Add file" → "Upload files"

3. Перетащить файлы или нажать "choose your files"

4. Заполнить сообщение коммита

5. Нажать "Commit changes"
```

Ограничение: загрузка через веб-интерфейс работает для файлов до 25 МБ. Для больших файлов — Git LFS или локальная работа.

## Редактирование в разных ветках

По умолчанию изменения вносятся в текущую ветку (обычно `main`). Но можно коммитить в новую ветку и сразу создать pull request:

```
1. Внести изменения в редакторе

2. В разделе "Commit changes" выбрать:
   "Create a new branch for this commit and start a pull request"

3. Ввести имя новой ветки
   Например: fix/update-readme

4. Нажать "Propose changes"

5. GitHub откроет форму создания pull request
```

Это полезно, если нельзя коммитить напрямую в `main` из-за защиты ветки.

## Быстрый редактор github.dev

Нажатие `.` на странице репозитория открывает `github.dev` — облачную версию VS Code прямо в браузере:

```
Возможности github.dev:
- Редактирование нескольких файлов
- Подсветка синтаксиса для всех языков
- Расширения VS Code (часть)
- Дерево файлов
- Поиск по всему репозиторию
- Коммит изменений через Source Control

Ограничения:
- Нельзя запускать код (нет терминала)
- Нельзя устанавливать зависимости
- Нет отладчика
```

Переключение между обычным просмотром и редактором: замените `github.com` на `github.dev` в адресной строке или нажмите `.`.

## GitHub Codespaces: полная среда разработки

Для более серьёзной работы без локального Git — GitHub Codespaces:

```
1. Нажать кнопку "Code" на странице репозитория

2. Выбрать вкладку "Codespaces"

3. Нажать "Create codespace on main"

4. GitHub запустит виртуальную машину с VS Code
   - Полный терминал
   - Возможность запускать код
   - Git предустановлен и настроен
   - Порты проброшены для тестирования веб-приложений
```

Codespaces — платный сервис (есть бесплатный лимит для личных аккаунтов). Подходит для разработки без настройки локального окружения.

## Ограничения веб-редактора

Веб-редактор GitHub подходит для простых правок, но имеет ограничения:

```
Что нельзя сделать через веб-редактор:
- Редактировать несколько файлов в одном коммите (только через github.dev)
- Работать с бинарными файлами (изображения, архивы)
- Запускать тесты или скрипты
- Работать с submodules
- Делать интерактивный rebase
- Разрешать сложные конфликты слияния
- Загружать файлы больше 25 МБ
```

Для всего этого нужна локальная работа с Git или Codespaces.

## Практические примеры

```
# Быстрые правки через браузер — хорошо подходит:
- Исправить опечатку в документации
- Обновить версию в package.json
- Добавить строку в .gitignore
- Написать README для нового проекта
- Ответить на review с небольшими правками

# Требует локального Git или github.dev:
- Рефакторинг нескольких файлов
- Добавление новой функции с тестами
- Разрешение конфликтов слияния
- Работа с большими бинарными файлами
```

## Просмотр истории изменений файла

После редактирования через веб-интерфейс можно посмотреть историю:

```
1. Открыть файл в репозитории

2. Нажать "History" (иконка часов) в правом верхнем углу

3. Просмотреть список коммитов для этого файла

4. Нажать на любой коммит для просмотра изменений
```

Или через командную строку:

```bash
# История конкретного файла
git log --oneline -- README.md

# Просмотр изменений в конкретном коммите
git show abc123 -- README.md
```

## Часто задаваемые вопросы

**Можно ли редактировать несколько файлов в одном коммите через браузер?** Стандартный редактор GitHub — только один файл за раз. Для нескольких файлов используйте github.dev (нажмите `.` на странице репозитория) или локальный Git.

**Как создать папку через веб-интерфейс?** GitHub не позволяет создать пустую папку. Создайте файл и укажите путь с папкой: `новая-папка/файл.txt`. Папка создастся автоматически.

**Можно ли отменить изменения, сделанные через веб-интерфейс?** Да, через `git revert` или создание нового коммита с правками. В самом браузере — через кнопку "Edit" в файле и ручное исправление.

**Чем github.dev отличается от GitHub Codespaces?** `github.dev` — это VS Code в браузере без терминала и возможности запускать код. Codespaces — полноценная виртуальная машина с терминалом, она платная (есть бесплатный лимит).

**Как коммитить сразу в новую ветку и создать pull request?** При сохранении изменений в редакторе выберите опцию "Create a new branch" вместо "Commit directly to main".

## Заключение

Веб-редактор GitHub удобен для быстрых правок документации и конфигурационных файлов. Для работы с несколькими файлами — используйте `github.dev` (нажмите `.`). Для полноценной разработки — локальный Git с [навигацией через Git Bash]({{< relref "navigaciya-git-bash" >}}) или GitHub Codespaces.
